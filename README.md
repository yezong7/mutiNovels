# mutiNovels — 长篇小说自动化创作系统

基于文件驱动的多项目长篇小说创作系统，运行在 Trae IDE 中，通过 `.md` 文件管理设定、大纲、进度和正文，配合四阶段写作流程和严格的一致性校验机制。

## 项目结构

```
mutiNovels/
├── .trae/
│   ├── rules/
│   │   ├── novelworkflow.md    ← 全局工作流规则（自动注入上下文）
│   │   └── templates.md        ← 文件模板规范（自动注入上下文）
│   └── rulelog.md              ← 规则变更日志 + 违规案例记录（按需读取，不自动注入）
├── authors/                     ← 作者风格库（全局共享，多项目引用）
│   ├── cold_hard_school.md     ← 冷硬派·东方克制
│   └── ...                     ← 可无限扩展
├── {项目名}/
│   ├── bible.md            ← 设定集（世界观/人物/地点/道具/势力/术语/伏笔索引）
│   ├── outline.md          ← 主体大纲（核心命题/故事结构/卷章规划/人物弧线/伏笔路线图/偏差记录）
│   ├── state.md            ← 写作进度（含阶段暂存内容）
│   ├── rules.md            ← 项目专属约束（文风/叙事/禁写）+ 引用作者风格 + 可选覆盖
│   ├── changelog.md        ← 变更日志
│   └── chapters/
│       ├── chapter_01.md
│       ├── chapter_02.md
│       └── ...
```

## 指令集

| 指令 | 触发口令 | 说明 |
|------|---------|------|
| 初始化 | `【初始化】` 或 `【初始化】项目名` | 创建项目目录、所有模板文件、首章空文件 |
| 加载上下文 | `【加载上下文】` 或 `【开始写作】` | 分层加载文件、总结当前处境、显示阶段和大纲位置、检查暂存恢复 |
| 存档下班 | `【存档下班】` | 写入章节文件→更新 state→更新 bible→更新 outline 偏差→追加 changelog |
| 查设定 | `【查设定】名字/物品` | 搜索 bible + 全部章节，汇总出场记录和伏笔关联 |
| 新建章节 | `【新建章节】` | 创建 chapter_XX.md，更新 state |
| 修订 | `【修订】chapter_02` | 局部修订或一致性修订，连锁更新 bible/outline/后续章节 |
| 回退 | `【回退】` 或 `【重新构思】`等 | 回退到当前阶段或上一阶段，清空暂存内容 |
| 切换风格 | `【切换风格】作者文件名/风格描述` | 切换项目引用的作者风格文件，支持创建新风格 |

## 四阶段写作流程

```
用户输入                助手输出                       确认机制
──────────────────────────────────────────────────────────────────
【构思】描述  ──→  3-5个方案（含大纲对齐+命题对齐）──→  用户选择方案
     │
     ▼
【草稿】      ──→  完整场景草稿（含偏差标注）        ──→  用户确认/修改
     │
     ▼
【正文】      ──→  定稿正文（含命题锚点标注）        ──→  用户确认
     │
     ▼
【润色】      ──→  润色全文 + 风格校准报告           ──→  用户确认
     │                                              （样本对比+12条清单）
     ▼
【存档下班】  ──→  写入文件 + 连锁更新               ──→  ✅ 存档完成
```

### 各阶段详情

| 阶段 | 推荐模型特性 | 助手核心职责 | 输出标注 |
|------|------------|------------|---------|
| 构思 | 创意型、大参数 | 冲突检测（人物/地点/道具）→ 输出3-5个方案，每个含核心冲突/涉及人物/情绪效果/衔接点/大纲对齐/命题对齐 | 无 |
| 草稿 | 速度型、长输出 | 基于选定方案快速写完整场景，标注与大纲偏差 | `[草稿待确认]` |
| 正文 | 精准型、逻辑强 | 基于确认草稿输出定稿，检查设定一致性+风格一致性+命题对齐 | `[正文待确认]` + 命题锚点 |
| 润色 | 风格型、语言敏感 | 打磨语言，执行12条风格校准清单（样本对比标注模式） | `[润色完成待存档]` |

## 确认机制

### 触发词白名单

只有以下措辞视为有效确认：

> **"确认""通过""存档""定稿""可以写入"**

### 非确认措辞（仅方向认可，不触发执行）

> "还行""不错""可以""行""嗯""好的""继续"

### 二次确认流程

```
用户说确认触发词
       │
       ▼
助手："收到确认，即将[执行动作]。确认执行？"
       │
       ▼
用户再次说确认触发词
       │
       ▼
助手执行动作

例外：【存档下班】指令无需二次确认（用户主动发起本身就是确认行为）
```

## 分层加载策略

`【加载上下文】` 时按三层加载，避免上下文窗口爆炸：

```
1. 必读层（每次都读）
   ├── state.md
   └── rulelog.md

2. 阶段层（按当前阶段决定）
   ├── 空闲  → outline 卷章+伏笔
   ├── 构思  → outline 全文 + bible 人物/势力
   ├── 草稿  → outline 卷章+偏差 + rules + bible 人物/地点/道具
   ├── 正文  → bible 全文 + rules 风格部分 + outline 卷章
   └── 润色  → rules 风格锚点+样本+校准清单

3. 章节层
   └── 当前章节最后 500 字

4. 暂存恢复
   └── 如有暂存内容 → 提示用户是否继续
```

### 进度可视化输出

加载上下文完成后，输出进度状态框：

```
╔════════════════════════════════════════╗
║  进度状态                              ║
╠════════════════════════════════════════╣
║  章节：chapter_03 (第3章)              ║
║  阶段：空闲                            ║
║  字数：11180                           ║
║  大纲：第一幕·寻踪 → 第二幕·对抗       ║
║  待回收伏笔：3条                       ║
╚════════════════════════════════════════╝
```

## 阶段跳转校验

用户可以跳过任意阶段或回退到前一阶段，但跳过时仍需执行核心检查：

| 跳过的阶段 | 必须执行的快速检查 | 输出格式 |
|-----------|-----------------|---------|
| 构思 | 对照 outline.md 标注"符合大纲/偏离大纲" | 一句话标注 |
| 草稿 | 对照 outline.md 卷章规划标注偏差 | 一句话标注 |
| 正文 | 对照 bible.md 检查设定一致性 + 对照 rules.md 风格锚点检查风格 | 一句话标注 |
| 润色 | 执行 rules.md 风格校准清单 | 逐条通过/未通过 |

输出示例：

> ⚡ 快速检查：大纲对齐——符合大纲 | 偏差——无偏差 | 设定一致性——无冲突 | 风格一致性——与锚点一致

## 阶段回退规则

**回退范围**：只可回退当前阶段或最多上一阶段

| 当前阶段 | 可回退至 |
|---------|---------|
| 构思 | 无 |
| 草稿 | 构思 |
| 正文 | 草稿、构思 |
| 润色 | 正文、草稿 |

**触发方式**：`【回退】` 或 `【重新构思】` `【重新草稿】` `【重新正文】`

**暂存清空**：回退后清空目标阶段及其后续阶段的暂存内容

## 构思阶段冲突检测

构思阶段输出方案前，自动检测以下冲突：

- 构思涉及的人物是否已在 `bible.md` 登记
- 构思涉及的地点/道具是否与 `bible.md` 已有描述矛盾
- 构思是否与 `outline.md` 已规划事件冲突

如发现潜在冲突，输出警告："⚠️ 检测到潜在冲突：[具体冲突点]"

## 作者风格系统

作者风格从项目级 `rules.md` 中解耦为独立的 `authors/xxx.md` 文件，实现：

- **多项目共享**：一个作者风格文件可被多个项目引用
- **动态切换**：修改 `rules.md` 第 7 节路径或使用【切换风格】指令即可切换
- **项目级覆盖**：`rules.md` 第 8 节可在作者风格基础上微调（如调整句子长度上限）

### 作者风格文件结构

```
authors/xxx.md
├── 1. 风格锚点（参考作家/流派/关键词/反面风格）
├── 2. 风格样本（动作A/情绪B/环境C/对话D）
└── 3. 风格校准清单（12条检查项）
```

### 风格加载优先级

```
作者风格文件（基础）→ rules.md 第 8 节覆盖项（叠加）→ 最终生效风格
```

### 切换风格示例

```
用户：【切换风格】cold_hard_school.md
助手：旧风格：无 → 新风格：冷硬派·东方克制
      关键差异：短句节奏/克制表达/冷感环境/极简对话
      ⚠️ 风格已切换。已写完的章节不受影响，后续写作将使用新风格。
```

## 风格校准流程（润色阶段）

润色时采用**样本对比标注**模式：

```
1. 从润色后正文中选取4段：
   ├── 动作段 → 与风格样本A并列对比
   ├── 情绪段 → 与风格样本B并列对比
   ├── 环境段 → 与风格样本C并列对比
   └── 对话段 → 与风格样本D并列对比

2. 逐条检查12项校准清单（✅/❌ + 具体位置 + 修改建议）

3. 输出风格校准报告
```

校准报告示例：

```
--- 风格校准报告 ---

【动作 vs 样本A】
正文：他闪身避开，拳头砸在墙上，指节发白。
样本：他侧身。刀光从左肩擦过，带起一线血。没退。
差异：正文节奏偏慢（1句 vs 3短句），缺少"动作-结果"的断句节奏。→ 建议拆分

【情绪 vs 样本B】
正文：他把胶囊放回口袋，手在发抖，眼睛湿了。
样本：他把空管放回口袋，和另外六管排在一起。七管。一个完整的人，装在口袋里。他站起来，膝盖没抖。
差异：正文直接写"手在发抖，眼睛湿了"，样本靠物件传达。→ 未通过，建议改为物件传达

校准清单：
✅ 句子平均长度 ≤15字
❌ 情绪靠动作/物件传达（第X段直接陈述情绪）
✅ 无廉价转折词
✅ 形容词不超标
✅ 对话每句 ≤20字
✅ 段落结尾有呼吸空间
✅ 环境描写服务情绪
✅ 动作段与样本A一致
❌ 情绪段与样本B一致（需调整）
✅ 环境段与样本C一致
✅ 对话段与样本D一致
```

## 伏笔管理（单一数据源）

```
outline.md 伏笔路线图 ──→ 唯一数据源（完整记录：伏笔/埋设章节/回收章节/回收方式/状态）
         │
         ▼ 引用
bible.md 伏笔索引 ──→ 仅快速查找（关键词+状态+指向outline）

新增伏笔：先写 outline → 再加 bible 索引行
回收伏笔：更新 outline 状态 → 同步更新 bible 索引
禁止：在 bible 索引中添加 outline 路线图中不存在的伏笔
```

## 跨会话恢复

`state.md` 中的"阶段暂存内容"字段保存每个阶段的中间产物：

| 阶段 | 暂存内容 |
|------|---------|
| 构思 | 用户选定的方案编号及核心冲突一句话 |
| 草稿 | 草稿核心内容摘要（3-5句话） |
| 正文 | 正文核心内容摘要（3-5句话） |
| 润色 | 润色版本核心改动点（2-3句话） |

下次 `【加载上下文】` 时，如暂存内容不为空，提示："上次会话停留在[阶段]阶段，有暂存内容。是否从暂存点继续？"

存档完成后，暂存内容清空为"无"。

## 修订流程

```
【修订】chapter_02
         │
         ▼
选择模式：局部修订 / 一致性修订
         │
         ▼
用户确认修订内容
         │
         ▼
连锁更新：
  ├── 修改章节文件
  ├── 检查 bible.md 影响 → 同步更新
  ├── 检查 outline.md 影响 → 同步更新
  └── 检查后续章节影响 → 列出影响清单
         │
         ▼
追加 changelog（修订格式）
         │
         ▼
✅ 修订完成

⚠️ 重大设定变更 → 提醒"建议检查所有后续章节"
⚠️ 修订不改变 state.md 的当前章节和当前阶段
```

## 核心文件说明

| 文件 | 用途 | 关键规则 |
|------|------|---------|
| bible.md | 世界观/人物/地点/道具/势力/术语/伏笔索引 | 一级分类不可增减；退场人物不删除只移动；伏笔索引仅引用 outline |
| outline.md | 核心命题/故事结构/卷章规划/人物弧线/伏笔路线图/偏差记录 | 伏笔唯一数据源；偏差只追加不删除；卷章状态：🔲待写/✏️写作中/✅已完成 |
| state.md | 当前进度/阶段/暂存内容 | 每次阶段切换必须更新；确认正文后标记"待存档正文：是" |
| rules.md | 文风/叙事/禁写/风格锚点/风格样本/风格校准清单 | 风格样本必须用户确认；校准清单12条润色时逐条执行 |
| changelog.md | 存档和修订的变更记录 | 每次存档/修订后追加，含 bible/outline 更新项 |
