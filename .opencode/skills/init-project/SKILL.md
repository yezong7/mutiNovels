# 初始化项目

当用户输入【初始化】或要求创建新项目时，执行以下完整流程。

## 执行步骤

### 1. 确认项目目录

- 询问用户项目名称（如用户已指定则直接使用）
- 项目名称规则：大写英文 + 数字，如 `TEST1`、`PROJECT2`
- 项目根路径为：`{工作目录}/{项目名}/`

### 2. 创建目录结构

```
{项目名}/
├── chapters/
├── bible.md
├── outline.md
├── state.md
├── rules.md
└── changelog.md
```

### 5. 写入 state.md

使用 Write 工具创建 `{项目名}/state.md`，将 `{项目名}` 替换为实际项目名：

```markdown
# 写作进度

- 项目目录：{项目名}/
- 当前章节：chapter_01.md
- 章节编号：1
- 当前阶段：空闲
- 累计字数：0
- 剧情进度：（尚未开始）
- 最后一句话：（无）
- 下一步计划：（无）
- 待存档正文：否
- 待存档内容摘要：无
- 阶段暂存内容：无
```

### 6. 写入 rules.md

使用 Write 工具创建 `{项目名}/rules.md`，将 `{项目名}` 替换为实际项目名：

```markdown

> 本文件定义项目的专属创作约束，与全局规则配合使用。
> 全局规则管工作流，本文件管文风与偏好。

## 1. 文风要求
- 叙事视角：
- 语言风格：
- 描写偏好：
- 叙事节奏：

## 2. 叙事约束
- 视角限制：
- 时间线处理：
- 场景切换规则：

## 3. 对话规则
- 对话占比上限：
- 对话风格：
- 对话标签偏好：

## 4. 禁写事项
- [列出禁止出现的写法]

## 5. 章节节奏
- 每章最低要求：
- 每章字数目标：
- 章末钩子：

## 6. 特殊约定
- [项目特有的约定]

## 7. 引用作者风格

> 指定本项目使用的作者风格文件。写作时自动加载该文件的风格锚点、样本和校准清单。
> 可随时切换为其他作者文件，实现风格动态配置。

- **当前作者**：（未指定）
- **切换方式**：修改本节路径即可，如 `authors/cold_hard_school.md` → `authors/poetic_realism.md`

> 如需在作者风格基础上做项目级微调，在此追加覆盖项。
> 覆盖项优先级高于作者文件。留空则完全使用作者文件定义。

（无覆盖 / 列出覆盖项）
```

### 7. 创建章节文件和变更日志

使用 Write 工具创建 `{项目名}/chapters/chapter_01.md`，内容为空。

使用 Write 工具创建 `{项目名}/changelog.md`，内容如下：

```markdown
# 变更日志

> 每次存档后自动追加记录。
```

### 8. 完成提示

输出以下信息：

```
✅ 项目 {项目名} 已初始化！

已创建文件：
  ├── bible.md          ← 请填写世界观、人物等基础设定
  ├── outline.md        ← 请填写核心命题、故事结构、卷章规划
  ├── state.md          ← 进度状态（自动维护）
  ├── rules.md          ← 项目专属创作约束
  ├── changelog.md      ← 变更日志（自动维护）
  └── chapters/
      └── chapter_01.md ← 第一章（空）

下一步：
1. 填写 bible.md 中的基础设定和 outline.md 中的大纲
   ——或告诉我你的故事构想，我来帮你填充
2. 在 rules.md 第 7 节指定作者风格文件路径
   ——如 authors/cold_hard_school.md，或告诉我你想要的风格，我来创建
```
