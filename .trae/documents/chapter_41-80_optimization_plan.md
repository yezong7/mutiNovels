# 计划：章节优化（41章及后续）

## 任务目标
按规则优化 project1 第41章及后续所有章节，每次只优化一章，优化时读取前后章节保持剧情连贯，剧情不大幅变动。

## 优化依据
- **项目约束**：`rules.md`（文风、叙事、对话、禁写、节奏等8类规则）
- **作者风格**：`authors/luan.md`（乱·宠魅型风格：系统化世界观、回合制战斗、量化成长、羁绊默契）
- **违规记录**：`rulelog.md`（已知的2个违规案例：构思/草稿/正文混淆、过度短句化）

## 优化检查清单

### 规则层面（rules.md）
- [ ] 叙事视角：第三人称有限视角，以林默为中心
- [ ] 对话占比 ≤30%，每句 ≤30字
- [ ] 禁写：模糊情绪标签、无信息对话、跳跃式成长、装饰性环境描写
- [ ] 每章 5000-9000 字
- [ ] 主角塑造：善良但不天真，理性克制，有具体目标

### 风格层面（luan.md）
- [ ] 设定量化：等级/阶段/防御级别/价值价格
- [ ] 战斗回合结构：技能名→施展→效果→结果
- [ ] 成长量化：当前状态→训练→新状态确认
- [ ] 情绪克制：用动作/物件传达，不直接标签
- [ ] 对话信息密集：时间+地点+对手+赌注+条件一次说完

### 违规规避（rulelog.md）
- [ ] 避免过度短句化（保持"短-中-长"节奏变化）
- [ ] 避免模糊情绪词（愤怒/恐惧/震惊→动作/眼神）

## 章节范围
- 当前状态：chapter_01 ~ chapter_80 已完成
- 待优化起始：chapter_41.md
- 待优化终止：chapter_80.md
- 共计：40章（41-80）

## 优化流程（每章）

1. **读取上下文**
   - 前一章（保持剧情连续性）
   - 后一章（确保不产生矛盾）
   - 当前章（待优化）

2. **逐项检查**
   - 对照 rules.md 8类规则
   - 对照 luan.md 风格校准清单（3.1-3.6）
   - 检查违规案例避免重复

3. **输出优化方案**
   - 标注具体问题位置（行号或段落）
   - 给出修改建议
   - 保持剧情不变，只优化表达

4. **等待用户确认**
   - 用户确认后执行修改
   - 每次只优化一章
