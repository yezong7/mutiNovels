# 文件模板规范

> 本文件定义 bible.md、state.md、outline.md、rules.md 的强制结构模板。
> 与 `novelworkflow.md` 配合使用——`novelworkflow.md` 定义工作流，本文件定义文件结构。

  - 回收伏笔时：更新 `outline.md` 状态为"✅ 已回收"，同步更新 `bible.md` 索引状态
  - **禁止**在 `bible.md` 索引中添加 `outline.md` 路线图中不存在的伏笔
  - "outline.md 伏笔关键词"列必须填写 `outline.md` 伏笔路线图第一列的完整关键词，用于快速定位，禁止填写"伏笔路线图"等无定位价值的占位文字

## 二、state.md 格式模板

```markdown
# 写作进度

- 项目目录：TEST1/
- 当前章节：chapter_01.md
- 章节编号：1
- 当前阶段：构思 / 草稿 / 正文 / 润色 / 空闲
- 累计字数：0
- 剧情进度：[一句话概括最近发生的事]
- 最后一句话：[复述最后写完的一句，方便下次无缝衔接]
- 下一步计划：[接下来打算写什么]
- 待存档正文：是 / 否
- 待存档内容摘要：[一句话描述待存档的正文内容，若无则填"无"]
- 阶段暂存内容：[当前阶段的中间产物，跨会话恢复用，格式见下方说明]
```

**更新规则**：

- 每次阶段切换必须更新"当前阶段"
- 用户确认正文后，将"待存档正文"改为"是"
- 存档完成后，将"待存档正文"改回"否"
- "累计字数"在每次存档后重新统计
- **阶段暂存内容**的格式与更新规则：
  - 构思阶段：暂存用户选定的构思方案编号及核心冲突一句话
  - 草稿阶段：暂存草稿的核心内容摘要（3-5 句话，覆盖场景/人物/转折）
  - 正文阶段：暂存正文的核心内容摘要（3-5 句话）
  - 润色阶段：暂存润色版本的核心改动点（2-3 句话）
  - 存档完成后，清空"阶段暂存内容"为"无"
  - 每次用户确认某阶段产出后、进入下一阶段前，更新暂存内容

## 三、章节文件命名与格式

- 命名规则：`chapter_XX.md`，XX 为两位数补零（01, 02, ..., 99）
- 每个章节文件开头格式：

```markdown
# 第X章 章节标题

[正文内容]
```

- 章节标题由用户在存档时指定，或由助手根据内容建议

## 五、rules.md 模板

`rules.md` 包含 6 节项目专属约束 + 1 节作者风格引用 + 1 节项目级覆盖：

```markdown
# {项目名} 项目创作约束

> 本文件定义项目的专属创作约束，与全局规则配合使用。
> 全局规则管工作流，本文件管文风与偏好。

## 1. 文风要求
- 叙事视角：
- 语言风格：
- 描写偏好：
- 叙事节奏：

## 2. 叙事约束
- 视角限制：
- 时间线处理：
- 场景切换规则：

## 3. 对话规则
- 对话占比上限：
- 对话风格：
- 对话标签偏好：

## 4. 禁写事项
- [列出禁止出现的写法]

## 5. 章节节奏
- 每章最低要求：
- 每章字数目标：
- 章末钩子：

## 6. 特殊约定
- [项目特有的约定]

## 7. 引用作者风格

> 指定本项目使用的作者风格文件。写作时自动加载该文件的风格锚点、样本和校准清单。
> 可随时切换为其他作者文件，实现风格动态配置。

- **当前作者**：`authors/xxx.md`
- **切换方式**：修改本节路径即可，如 `authors/cold_hard_school.md` → `authors/poetic_realism.md`

## 8. 项目级风格覆盖（可选）

> 如需在作者风格基础上做项目级微调，在此追加覆盖项。
> 覆盖项优先级高于作者文件。留空则完全使用作者文件定义。

（无覆盖 / 列出覆盖项）
```

**追加规则**：

- 第 1-6 节为项目专属约束，与作者风格无关（文风偏好、叙事规则、禁写等）
- 第 7 节引用作者风格文件，是风格锚点/样本/校准清单的**唯一来源**
- 第 8 节为可选覆盖，允许项目在作者风格基础上微调（如调整句子长度上限、追加禁用词等）
- **禁止**在 rules.md 中重复定义作者文件已有的风格锚点/样本/校准清单
- 切换作者风格只需修改第 7 节路径，无需修改其他文件
- 润色阶段加载风格时：先读取作者文件，再叠加项目级覆盖（如有）

## 六、模型推荐参考

| 阶段 | 推荐模型特性   | 说明           |
| -- | -------- | ------------ |
| 构思 | 创意型、大参数  | 需要发散思维和丰富联想  |
| 草稿 | 速度型、长输出  | 需要快速产出大量文字   |
| 正文 | 精准型、逻辑强  | 需要严格一致性和细节校准 |
| 润色 | 风格型、语言敏感 | 需要修辞能力和文字美感  |

用户可在任意阶段手动切换模型，此表仅为推荐参考。

## 七、作者风格文件模板

> 完整模板见 `authors/_template.md`，以下为结构概览和追加规则。

作者风格文件存放在 `authors/` 目录下，可被多个项目引用。每个文件定义一种完整的写作风格。

### 结构概览

```
authors/xxx.md
├── 1. 风格锚点
│   ├── 1.1 参考作家与对标点（作家/作品/学什么/不学什么）
│   ├── 1.2 风格流派与定位（流派/位置/与相邻流派的边界）
│   ├── 1.3 风格关键词（核心词 + 质感描述）
│   ├── 1.4 反面风格（反面倾向/具体表现/替代方案）
│   ├── 1.5 叙事节奏特征（句子/段落/场景节奏）
│   ├── 1.6 情感表达方式（传达路径/克制度/爆发时机）
│   ├── 1.7 意象与感官偏好（偏好意象/感官优先级/使用规则）
│   └── 1.8 留白策略（必须写/绝不写/留白位置）
├── 2. 风格样本（8 类，每类附锚点解析）
│   ├── A 动作场景
│   ├── B 情绪场景
│   ├── C 环境描写
│   ├── D 对话
│   ├── E 过渡与转场
│   ├── F**: 《宠魅》by 乱
> **风格核心**: 系统化世界观 + 回合制战斗 + 量化压力传达 + 信息 高潮/爆发点
│   ├── G 静默/日常
│   └── H 回忆/闪回
└── 3. 风格校准清单（5 类 20 条）
    ├── 3.1 语言层面（5 条）
    ├── 3.2 情感层面（3 条）
    ├── 3.3 对话层面（3 条）
    ├── 3.4 意象与环境层面（3 条）
    └── 3.5 样本对比校准（8 条，A-H 各一条）
```

### 追加规则

- 文件命名：小写英文 + 下划线，如 `cold_hard_school.md`、`poetic_realism.md`
- 模板文件：`authors/_template.md` 为空模板，创建新风格文件时基于此模板填充
- 风格样本必须由用户提供或确认，助手不得自行编造
- 风格样本每段 80-150 字（对话样本 5-8 轮），必须附带锚点解析
- 风格校准清单的具体数值（句子长度、心理描写上限等）由用户根据风格需要设定
- 一个作者文件可被多个项目同时引用
- 修改作者文件会影响所有引用该文件的项目（项目级覆盖不受影响）
- 8 类样本不要求全部填充，但至少必须包含 A-D 四类核心样本；E-H 为扩展样本，按需添加
