# 小说创作工作流规范

## 一、核心规则

1. 未读 `state.md` + `outline.md` 禁止续写正文
2. 修改内容必须直接写入 `.md` 文件，不只在对话输出
3. 每次操作后必须重写 `state.md`
4. **未经用户确认不得写入章节文件**。确认词白名单：确认/通过/存档/定稿/可以写入。非确认词：还行/不错/可以/行/嗯/好的/继续。【存档下班】例外，直接执行。
5. 写作前必须读 `rules.md` + 对应 `authors/xxx.md`
6. 每次写作前读 `changelog.md`。Memory 与 .md 冲突时以 .md 为准。
7. 未读 `outline.md` 禁止进入构思
8. 润色必须执行风格校准清单（叠加 rules.md §8 覆盖）

## 二、四阶段流程

```
构思 → 草稿 → 正文 → 润色
```

所有阶段在对话中输出，不写入文件。用户确认后才进入下一阶段。

### 构思【构思】

读 bible/state/outline + 前三章。执行冲突检测（人物/地点/道具/大纲）。输出 3-5 个方案，每个含：核心冲突、转折、人物、情绪效果、衔接点、**大纲对齐标注**、**命题对齐标注**。

### 草稿【草稿】

基于确认方案写完整场景。覆盖：场景描写/对话/动作/心理。标注与大纲偏差。末尾标 `[草稿待确认]`。

### 正文【正文】

基于确认草稿输出定稿。执行 **7 点一致性检查**：新人物登记、人物状态、地点、道具、时间线、术语、与已有章节无矛盾。检查风格一致性 + 命题对齐。末尾标 `[正文待确认]`。

### 润色【润色】

语言打磨 + **风格校准**（样本对比 A-D，校准清单逐条判定）。末尾标 `[润色完成待存档]`。

## 三、指令集

> 【初始化】→ skills/init-project/SKILL.md
> 【加载上下文】→ skills/load-context/SKILL.md
> 【新建章节】→ skills/new-chapter/SKILL.md

### 【存档下班】

1. 检查"待存档正文"=是
1.5 **字数验证**：统计字数，对照 rules.md §5 最低要求。不足则警告差额，用户确认可跳过
2. 写入 chapters/chapter_XX.md
3. 重写 state.md
4. 追加 bible.md（新增人物/地点/道具/势力）
5. 追加 outline.md 偏差
6. 更新 state.md 剧情进度（大纲预设 + 实际摘要）
7. 追加 changelog.md：写入章节/当前章节/剧情进度/最后一句/下一步/bible更新/outline偏差

### 【查设定】名字

读 bible.md + 全章搜索 + outline 伏笔路线图。汇总出场记录、当前状态、关联伏笔。矛盾高亮。

### 【修订】章节

读 state/目标章节/bible/outline。选模式（局部/一致性）。用户确认后执行：修改章节 → 同步 bible → 同步 outline → 列出后续章节影响清单 → **用户确认后**才执行后续修订。重大设定变更提醒检查所有后续章节。追加 changelog.md（修订格式）。修订不改变 state.md 的当前章节和阶段。

### 【切换风格】文件名

检查 authors/ 目录。更新 rules.md §7。输出新旧对比。提醒已写章节不受影响。追加 changelog.md（风格切换格式）。
