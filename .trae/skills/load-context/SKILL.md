# 加载上下文

当用户输入【加载上下文】、【开始写作】或要求恢复写作时，执行以下分层加载流程。

## 前置条件

- 必须已知当前活跃项目（由用户指定或从上次会话推断）
- 如果不确定项目，先询问用户

## 执行步骤

### 1. 必读层加载

使用 Read 工具读取以下文件：

- `{项目}/state.md` —— 获取当前阶段、章节、进度
- `.trae/rulelog.md` —— 获取违规案例（仅此文件，不含 rulelog_history.md）
- `{项目}/changelog.md` —— 获取最近 3 次变更记录

### 2. 前置章节层加载（新增）

**目的**：确保剧情连续性，防止写偏

根据 `state.md` 中的"章节编号"字段，加载前置章节：

| 当前章节编号 | 加载范围 |
|-------------|---------|
| 1 (第一章) | 无前置章节 |
| 2 | chapter_01 全文 |
| 3-5 | 前 2 章全文 + 前 3 结尾摘要 |
| 6+ | 前 3 章全文 + 前 4-5 章结尾摘要 |

**结尾摘要格式**：每章最后 300 字 + 一句话剧情总结

**章节编号获取**：从 `state.md` 的"章节编号"字段读取数字值

### 3. 阶段层加载

根据 `state.md` 中的"当前阶段"字段，决定读取范围：

| 当前阶段 | 必须读取 | 按需读取 |
|---------|---------|---------|
| 空闲 | `outline.md` 的卷章规划 + 伏笔路线图 | `bible.md` 世界观 |
| 构思 | `outline.md` 全文、`bible.md` 人物 + 势力、前置章节层 | `rules.md` |
| 草稿 | `outline.md` 卷章规划 + 偏差、`rules.md`、`bible.md` 人物 + 地点 + 道具、前置章节层 | 当前章节全文 |
| 正文 | `bible.md` 全文、作者风格文件风格锚点 + 风格样本、`outline.md` 卷章规划、前置章节层 | 当前章节全文 |
| 润色 | 作者风格文件全文（风格锚点 + 风格样本 + 风格校准清单）+ `rules.md` 第 8 节覆盖项、当前章节全文 | `outline.md` 主题意象 |

**作者风格文件路径**：从 `rules.md` 第 7 节获取，格式为 `authors/xxx.md`

**按需读取原则**：只有当用户提问涉及该内容时才读取，不要预先加载

### 4. 上一章衔接分析（新增）

如果当前章节编号 > 1，执行以下分析：

**读取上一章结尾**：
- 识别上一章最后一段的情绪/悬念
- 提取上一章的"章末钩子"（悬念句/转折点/未解决冲突）
- 记录上一章结尾在场人物及其状态

**输出衔接提示**：
```
【上一章衔接】
- 章末钩子：[上一章最后留下的悬念/转折]
- 在场人物：[上一章结尾在场的人物及其状态]
- 未解决冲突：[上一章遗留的待解决问题]
- 本章开头建议：[自然承接上一章的建议]
```

### 5. 当前章节层加载

根据 `state.md` 中的"当前章节"字段，加载当前章节：

- **章节编号 ≤ 3**：加载全文
- **章节编号 > 3 且字数 > 3000**：加载最后 500 字 + 开头 200 字（保持首尾衔接）

如果章节文件为空或不存在，跳过此步。

### 6. 剧情连续性检查（新增）

对比前置章节与当前状态，输出连续性检查报告：

```
【剧情连续性检查】
✅/❌ 人物状态连续性：[检查在场人物状态是否与上一章结尾一致]
✅/❌ 时间线连续性：[检查时间是否自然推进，无跳跃或倒退]
✅/❌ 场景连续性：[检查地点是否合理过渡]
✅/❌ 伏笔状态：[检查本章前已埋设的伏笔是否在正确回收阶段]
⚠️ 如发现问题：列出具体冲突点
```

### 7. 暂存恢复检查

检查 `state.md` 中"阶段暂存内容"字段：

- 如果不为"无"，输出提示：`"上次会话停留在[当前阶段]阶段，有暂存内容：[暂存内容摘要]。是否从暂存点继续？"`
- 如果为"无"，继续下一步

### 8. 生成上下文摘要

**绝对不要输出原文！** 只用 5-6 句话总结：
- 剧情走向（最近 3 章发生了什么）
- 当前处境（人物在哪里、面临什么）
- 在场人物及其状态
- 未解决冲突/悬念
- 情绪基调
- 下一步预判（大纲规划）

### 9. 大纲定位与伏笔状态

从 `outline.md` 的卷章规划中：

- 标注当前章节在大纲中的位置
- 列出待回收伏笔数量及内容摘要
- 列出当前阶段应该埋设的伏笔（如有）

### 10. 进度可视化输出

输出以下格式的进度框：

```
╔════════════════════════════════════════╗
║  进度状态                              ║
╠════════════════════════════════════════╣
║  章节：chapter_XX (第X章)              ║
║  阶段：[当前阶段]                      ║
║  字数：[累计字数]                      ║
║  大纲：[当前幕名] → [下一幕名]         ║
║  待回收伏笔：[数量]条                  ║
║  前置章节已加载：[章节列表]            ║
╚════════════════════════════════════════╝
```

字段来源：
- 章节/字数：`state.md`
- 阶段：`state.md`
- 大纲：`outline.md` 故事结构，取当前章节所属幕名
- 待回收伏笔：`outline.md` 伏笔路线图中状态为 🔴 的行数
- 前置章节：本次加载的章节列表

### 11. 完成提示

输出：`"上下文已就绪，已加载前置章节确保剧情连续。今天怎么发展？"`

## 注意事项

- 读取文件时使用 Read 工具，不要使用 Bash 命令
- 如果某个文件不存在，输出警告并跳过，不要中断流程
- 如果 `state.md` 中阶段为"空闲"且暂存为"无"，说明是新会话，正常加载即可
- 润色阶段优先加载作者风格文件 + `rules.md` 第 8 节覆盖项，其余延后
- **前置章节加载是防止写偏的关键，不可跳过**
- 如发现剧情连续性问题，必须在输出中高亮提醒用户
