# 加载上下文

当用户输入【加载上下文】、【开始写作】或要求恢复写作时，执行以下分层加载流程。

## 前置条件

- 必须已知当前活跃项目（由用户指定或从上次会话推断）
- 如果不确定项目，先询问用户

## 执行步骤

### 1. 必读层加载

使用 Read 工具读取以下文件：

- `{项目}/state.md` —— 获取当前阶段、章节、进度
- `.trae/rulelog.md` —— 获取违规案例（仅此文件，不含 rulelog_history.md）

### 2. 阶段层加载

根据 `state.md` 中的"当前阶段"字段，决定读取范围：

| 当前阶段 | 必须读取 | 按需读取 |
|---------|---------|---------|
| 空闲 | `outline.md` 的卷章规划 + 伏笔路线图 | `bible.md` 世界观 |
| 构思 | `outline.md` 全文、`bible.md` 人物 + 势力 | `rules.md` |
| 草稿 | `outline.md` 卷章规划 + 偏差、`rules.md`、`bible.md` 人物 + 地点 + 道具 | `changelog.md` |
| 正文 | `bible.md` 全文、作者风格文件风格锚点 + 风格样本、`outline.md` 卷章规划 | `outline.md` 伏笔路线图 |
| 润色 | 作者风格文件全文（风格锚点 + 风格样本 + 风格校准清单）+ `rules.md` 第 8 节覆盖项 | `outline.md` 主题意象 |

**作者风格文件路径**：从 `rules.md` 第 7 节获取，格式为 `authors/xxx.md`

**按需读取原则**：只有当用户提问涉及该内容时才读取，不要预先加载

### 3. 章节层加载

根据 `state.md` 中的"当前章节"字段，使用 Read 工具读取 `chapters/` 下对应文件的最后 500 字（使用 offset 参数跳过前面内容）。

如果章节文件为空或不存在，跳过此步。

### 4. 暂存恢复检查

检查 `state.md` 中"阶段暂存内容"字段：

- 如果不为"无"，输出提示：`"上次会话停留在[当前阶段]阶段，有暂存内容：[暂存内容摘要]。是否从暂存点继续？"`
- 如果为"无"，继续下一步

### 5. 生成上下文摘要

**绝对不要输出原文！** 只用 3-4 句话总结：
- 当前处境（人物在哪里、面临什么）
- 在场人物
- 情绪基调

### 6. 大纲定位

从 `outline.md` 的卷章规划中，标注当前章节在大纲中的位置，并列出待回收伏笔数量。

### 7. 进度可视化输出

输出以下格式的进度框：

```
╔════════════════════════════════════════╗
║  进度状态                              ║
╠════════════════════════════════════════╣
║  章节：chapter_XX (第X章)              ║
║  阶段：[当前阶段]                      ║
║  字数：[累计字数]                      ║
║  大纲：[当前幕名] → [下一幕名]         ║
║  待回收伏笔：[数量]条                  ║
╚════════════════════════════════════════╝
```

字段来源：
- 章节/字数：`state.md`
- 阶段：`state.md`
- 大纲：`outline.md` 故事结构，取当前章节所属幕名
- 待回收伏笔：`outline.md` 伏笔路线图中状态为 🔴 的行数

### 8. 完成提示

输出：`"上下文已就绪，今天怎么发展？"`

## 注意事项

- 读取文件时使用 Read 工具，不要使用 Bash 命令
- 如果某个文件不存在，输出警告并跳过，不要中断流程
- 如果 `state.md` 中阶段为"空闲"且暂存为"无"，说明是新会话，正常加载即可
- 润色阶段优先加载作者风格文件 + `rules.md` 第 8 节覆盖项，其余延后
